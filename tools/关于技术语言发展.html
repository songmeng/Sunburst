java 作为强类型编译语言, 明显编码的时候敲更多代码, 但是因为有编译时类型检测, 运行时的出错率更低, 同时执行效率也更高. 但是这些都是以开发效率为代价的. 所以如果你看中运行效率, 可能 java 更合适. 如果你更看重灵活和开发效率, php 可能更合适. 但是 "效率" 这种东西并不是绝对的, 因为影响 "效率" 的因素实在是太多了. 至于学习曲线, php 更容易上手做项目. 但是这并不代表 php 就是 "肤浅", 这只是因为 php 本身特性导致而已. 那些动不动就说 php 写了几年只能写什么什么东西的, 给他 java 可能也写不出什么东西. php 对面向对象的支持非常全面而且简洁. 包括文档, 社区以及包管理系统现在都做的很好. php7 的出现, 也让其运行性能有了大幅度提升.
