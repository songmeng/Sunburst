问题描述：android项目如果引用有其他的项目moduleLib,moduleLib中如果引用有第三发C.jar，主项目在引用问题描述：android项目如果引用有其他的项目moduleLib可能时会出现无法引用C.jar的问题。

解决方法不复杂，主要是因为不熟悉implementation和api在用法上的区别：

implementation：依赖的jar或aar或library仅限模块内部使用

api：相关依赖可以在本module被其他module依赖时直接使用

说起这个问题，还是因为gradle升级引起的

Warning:Configuration 'compile' is obsolete and has been replaced with 'implementation'. It will be removed at the end of 2018

感觉这种升级有点简单暴力，太没有大厂的风格了

WARNING: DSL element 'DexOptions.incremental' is obsolete and will be removed at the end of 2018 WARNING: API 'variant.getJavaCompile()' is obsolete and has been replaced with 'variant.getJavaCompileProvider()'. It will be removed at the end of 2019.。
